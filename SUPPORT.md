# Command Support

Use the public support guide first:

- [Support](https://galbutnotgirl.github.io/command/support.html)
- [Troubleshooting](https://galbutnotgirl.github.io/command/troubleshooting.html)
- [Install help](https://galbutnotgirl.github.io/command/install.html)
- [Bundled Markdown docs](docs/SUPPORT.md)
- [Latest Alpha release](https://github.com/galbutnotgirl/command/releases/latest)

Before filing a bug, open **Command -> Settings -> About -> Copy Diagnostic Info**, review copied diagnostics for sensitive content, then include the relevant lines.

File bugs with the [Bug report template](https://github.com/galbutnotgirl/command/issues/new?template=bug_report.md).

Request non-bug workflow, trigger, destination, docs, or release improvements with **Command -> Settings -> About -> Request Feature** or the [Feature request template](https://github.com/galbutnotgirl/command/issues/new?template=feature_request.md).

For vulnerabilities, exposed secrets, private logs, or sensitive diagnostic output, use **Command -> Settings -> About -> Private Security Report** or [Security Policy](SECURITY.md) instead of a public issue.
